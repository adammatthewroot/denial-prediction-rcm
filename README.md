# Prevention-First Claim Scoring
### A Machine Learning Approach to Predicting Medicare Claim Denials Before Submission

**Adam M. Root** | Executive Product Leader | Master's Candidate, Artificial Intelligence

---

## Overview

Healthcare claim denials cost U.S. providers an estimated $262 billion annually. This research demonstrates a prevention-first approach: a machine learning system that scores individual claims for denial risk *before submission*, identifies the most probable denial reason (CARC code), and routes high-risk claims for targeted intervention.

**Key results:**
- 88.9% accuracy on held-out patient-level test set (threshold = 0.651)
- AUC-ROC: 0.73
- Projected denial rate reduction: 28–32% (CARC-code-specific correctability model)
- Projected clean claim rate improvement: 3+ percentage points

## Data

Claims data is generated to match the **CMS DE-SynPUF Carrier Claims schema** (2008–2010). The SynPUF was released by CMS specifically for developing claims-data applications. All field names and value conventions match the official CMS codebook.

**On denial labels:** SynPUF provides the 837 (claim) side of the transaction but not the 835 (remittance advice). Denial labels are synthesized using published denial rate statistics from CMS, AMA, and KFF sources. See the notebook Section 2.2 for full methodology and limitations.

---

## Models

Two sequential XGBoost models:

1. **Binary model** — Denial probability score for any claim at submission time
2. **Reason model** — Most likely CARC denial code for high-risk claims

## Key Methodology Decisions

| Decision | Rationale |
|---|---|
| Patient-level train/test split | Prevents beneficiary correlation leakage |
| Submission-time features only | Prevents temporal leakage; matches deployment reality |
| XGBoost | Standard for tabular claims data; interpretable via SHAP |
| CARC-specific correctability | More defensible than flat assumption for impact projection |
| Stochastic label noise | Prevents model from memorizing label-generation formula |

---

## Limitations

The primary limitation is synthetic labels. Feature importances reflect the label generation formula rather than real payer adjudication patterns. The model architecture and evaluation methodology transfer to real 835 remittance data; specific feature weights require revalidation. See Section 9.2 of the notebook for full discussion.
